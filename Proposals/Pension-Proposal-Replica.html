01
Introduction
Newtuple proposes a practical automation layer for pension contribution processing. The solution uses Microsoft Power Automate where it fits best, mailbox monitoring, attachment handling, notifications, and a controlled AI agent and validation API where interpretation, comparison, and auditability need stronger ownership.
The client receives monthly contribution files from multiple employers. These files contain employee contributions, new joiners, exits, retirements, and other membership changes. Today, the work depends on manual review of emails, attachments, prior submissions, validation rules, employer comments, and final upload into the pension administration system.
The proposed engagement focuses on removing this manual load without creating a large workflow platform. The design keeps the existing employer portal intact, introduces an API-accessible agent and validation service, and uses Power Automate for mailbox orchestration and process routing.
This proposal is prepared as a budgetary estimate . The final scope, licensing, API behaviour, rule library, and commercial terms should be confirmed during discovery. Estimates carry a ±30% variance until then.

04
Technical Architecture
System view, indicative
This view shows how the logical flow is realised across Microsoft 365, the Power Automate integration layer, the custom application on Azure, and the downstream pension administration platform. All cross-component access is over HTTPS, with audit logging at every hop.
INTEGRATION LAYER
Microsoft 365 mailbox monitored by Power Automate. Triggers orchestrate intake, call APIs, and emit status notifications back to employers.
SYSTEM OF RECORD
FastAPI backend with business logic and validation. PostgreSQL holds submissions, employers, audit trail, errors, and file versions. React dashboard for operations.
STORAGE & DOWNSTREAM
Azure Blob or SharePoint as the contribution file repository. Upload to the pension administration platform via API or SFTP, subject to integration availability.
Security & Operating Controls
●Authentication , role-based access across dashboard, validation API, and PAS endpoints.
●Transport security , HTTPS everywhere including internal API calls.
●Audit logging , every submission, decision, error, notification, and upload recorded with timestamp and actor.
●Secret management , credentials held in environment variables or a managed vault, never in source.
●Hosting , custom application runs on an Azure VM with Nginx fronting the dashboard and API endpoints.
●Conditional integration , PAS posting depends on confirmed API or SFTP availability during discovery.

Our Delivery Advantage
Newtuple's advantage in this engagement is practical delivery discipline. The proposed design avoids over-engineering, keeps the client's existing portal intact, and puts the highest-risk logic in a service layer that can be tested, audited, and changed without rewriting process flows.
●Clear separation between workflow orchestration and business validation logic.
●Agent-based interpretation only where it matters, employer comments and instructions in mailbox-led submissions.
●A reusable validation API that serves both mailbox and portal flows.
●Audit-first processing so every file, decision, error, notification, and upload status can be traced.
●A delivery model sized for a budgetary decision now, with discovery used to confirm the final build path.
